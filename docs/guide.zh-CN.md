# 使用指南

[English](guide.md) | 简体中文

本指南从零开始带你完成 OpenClaw Connector 的配置和使用。不需要 SSH 隧道或 AI Agent 基础设施的相关知识。

## 开始之前

使用 OpenClaw Connector 之前，你需要准备好两件事：

### 1. 一台运行 OpenClaw 网关的服务器

OpenClaw Connector 是一个**客户端**——它连接到运行在远程 Linux 服务器上的 [OpenClaw](https://github.com/openclaw/openclaw) 网关。你（或你的团队管理员）必须先安装并配置好网关。

> 如果还没有搭建网关，请参考 [OpenClaw 文档](https://github.com/openclaw/openclaw)。

### 2. 能通过 SSH 连接到服务器

你的本地电脑必须能通过 SSH 连接到服务器。验证方法：

```bash
ssh 你的用户名@服务器IP
```

如果这个命令能成功登录服务器，就没问题。如果不行，你需要：
- 向管理员获取服务器 IP 地址和 SSH 登录凭证
- 配置 SSH 密钥对（参考 [GitHub 的 SSH 指南](https://docs.github.com/zh/authentication/connecting-to-github-with-ssh/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent)）

## 配置参数说明

打开应用后，你会看到一个连接配置表单。下面逐项解释每个字段的含义：

### 连接设置

| 参数 | 是什么 | 怎么找到它 | 示例 |
|------|--------|-----------|------|
| **主机地址** | 你的 Linux 服务器的 IP 地址或域名 | 问你的服务器管理员，或查看云服务商控制台 | `203.0.113.50` 或 `my-server.example.com` |
| **用户** | 登录服务器的 SSH 用户名 | 和你用 `ssh 用户名@主机` 时的用户名一样 | `root`、`ubuntu`、`deploy` |
| **密钥路径** | 你 Mac 上的 SSH 私钥路径 | 通常在 `~/.ssh/` 目录下。不确定的话，在终端运行 `ls ~/.ssh/` 查看 | `~/.ssh/id_ed25519` 或 `~/.ssh/id_rsa` |
| **Gateway Token** | OpenClaw 网关的认证令牌 | 在服务器上的网关配置文件中查找 `gateway.auth.token` 字段 | 一个长字符串，类似 `sk-abc123...` |

### 端口设置

| 参数 | 是什么 | 默认值 | 什么时候需要改 |
|------|--------|--------|---------------|
| **远程端口** | OpenClaw 网关在服务器上监听的端口 | `18789` | 仅当管理员配置了不同的端口时 |
| **本地端口** | 通过 SSH 隧道映射到本地的端口 | `18789` | 仅当该端口已被本地其他程序占用时 |

> **提示：** 大多数情况下，保持两个端口一致即可。应用会创建一条 SSH 隧道，把服务器上的远程端口映射到你的本地，这样你就能像访问本地服务一样访问远端网关。

### 其他设置

| 参数 | 是什么 | 默认值 |
|------|--------|--------|
| **节点名称** | 给你的本地机器起个名字，AI Agent 会看到这个名字 | 自动生成 |

## 连接步骤

1. **填写配置** — 输入服务器地址、用户名、密钥路径和 Gateway Token
2. **点击「重新连接」** — 应用会通过 SSH 隧道连接到你的服务器
3. **查看状态** — 状态徽标应显示绿色的「已连接」
4. **打开管理控制台**（可选）— 点击「管理控制台」在浏览器中访问网关的 Web 界面

## 浏览器自动化 (CDP)

这是一个**可选功能**，让 AI Agent 能控制你本地的 Chrome 浏览器。

### 什么是 CDP？

CDP（Chrome DevTools Protocol）是浏览器开发者工具使用的底层协议。启用后，AI Agent 可以：
- 打开网页
- 点击按钮、填写表单
- 截图
- 提取页面内容

### CDP 参数

| 参数 | 是什么 | 默认值 |
|------|--------|--------|
| **CDP 端口（本地）** | Chrome 监听自动化指令的端口 | `9222` |
| **远程映射端口** | 映射到服务器的端口，让 Agent 能访问你的浏览器 | `19222` |

### 使用方法

1. 点击「启动浏览器」— Chrome 会以专用配置打开一个独立窗口
2. Agent 现在可以通过 CDP 控制这个 Chrome 窗口了

> **注意：** 这会打开一个独立的 Chrome 实例，使用单独的用户配置。不会影响你正在使用的 Chrome 浏览器。

## 会话通知

连接成功后，你可以通知 AI Agent 你的本地节点已上线：

1. 在「会话注入」卡片中展开一个 Agent
2. 点击某个会话旁边的「通知」按钮 — Agent 会收到一条包含你节点信息的系统消息
3. Agent 就可以在你的本地机器上执行命令了

## 常见问题

### 「连接失败」

- 先验证 SSH 能不能通：在终端运行 `ssh -i ~/.ssh/你的密钥 用户名@主机地址`
- 检查应用中的密钥路径是否和你实际的 SSH 密钥位置一致
- 确保服务器的 SSH 端口（22）从你的网络可以访问

### 「找不到 Gateway Token」

SSH 登录到服务器，查看 OpenClaw 网关的配置文件，找到 `gateway.auth.token` 或类似字段。不确定的话问你的管理员。

### 「状态显示已连接但控制台打不开」

- 确认本地端口没有被其他应用占用
- 尝试把本地端口改成别的数字（比如 `18790`）

### 「浏览器启动不了」

- 确认你的 Mac 上安装了 Google Chrome
- 如果 Chrome 已经在运行，CDP 浏览器会作为独立实例打开——这是正常的
